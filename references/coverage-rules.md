# 标品覆盖规则

## 覆盖方式

| 覆盖方式 | 系数范围 | 说明 |
| --- | ---: | --- |
| 标品覆盖 | 0.1-0.3 | 标品已可用，仅需配置、少量文案或权限适配 |
| 部分覆盖 | 0.4-0.7 | 标品有主体能力，但流程、字段、页面或接口需要二开 |
| 全部定开 | 1.0 | 标品无对应能力，需要完整开发 |
| 待确认 | 1.0-1.3 | 覆盖情况未知，先按较保守估算并列入确认问题 |

## 判断依据

- `文档明确`: 需求文档直接说明标品覆盖、部分覆盖或定开。
- `用户确认`: 用户在对话中确认覆盖范围。
- `行业知识库推断`: 根据行业常规模块判断，但必须标注待确认。
- `待确认`: 没有可靠依据。

## 使用要求

凡是由行业知识库推断出的覆盖方式，都必须写入 `客户确认问题`。不要把推断当成已确认事实。
