# 风险规则

## 风险等级

| 风险等级 | 系数范围 | 判断 |
| --- | ---: | --- |
| 低 | 1.0 | 需求明确、流程简单、无复杂外部依赖 |
| 中 | 1.1-1.2 | 存在少量待确认、多端适配、一般第三方接口 |
| 高 | 1.3-1.5 | 需求边界不清、旧系统改造、复杂集成、支付对账、复杂权限或报表 |

## 常见风险类别

- 需求不清：功能边界、验收口径、流程分支未明确。
- 数据迁移：旧系统结构未知、历史数据清洗、字段映射。
- 第三方接口：接口文档缺失、联调窗口不确定、外部系统不稳定。
- 权限复杂：角色权限、数据权限、组织隔离、门店隔离、项目隔离。
- 报表复杂：统计口径、导出、图表、定时报表、历史数据回算。
- 支付对账：退款、分账、对账、发票、异常订单处理。
- 多端适配：iOS、Android、H5、小程序、PC Web 交互和审核差异。
- 合规审计：隐私、等保、操作留痕、电子合同、财务留存。

## 隐藏范围检查

评估时主动检查：

- 登录注册、组织架构、角色权限、数据权限
- 消息通知、短信、微信模板、App 推送
- Excel 导入导出、错误校验、模板下载
- 操作日志、审批记录、数据变更记录
- 部署、上架、应用市场审核、培训、文档、售后支持
