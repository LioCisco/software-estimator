# Buff 规则

Buff 是基础工时之外的预留时间，必须单独列出比例、人天和原因。不要把 Buff 隐藏在基础工时、风险系数或报价系数里。

## 默认 Buff 范围

| Buff 比例 | 适用情况 |
| ---: | --- |
| 0%-5% | 需求清晰、标品覆盖高、无复杂第三方集成 |
| 5%-10% | 有部分待确认需求、多端适配、少量第三方接口 |
| 10%-20% | 需求边界不完整、存在数据迁移、权限复杂、报表复杂、支付/退款/对账 |
| 20%-30% | 大量需求待确认、旧系统改造、复杂业务流、强依赖外部接口、验收口径不明确 |

## 常见 Buff 原因

- 多端适配范围未完全确认。
- 退款、支付、对账流程存在业务不确定性。
- 旧系统数据结构未知，迁移成本需预留。
- 第三方接口文档未提供。
- 审批流节点和权限边界待确认。
- 报表统计口径未明确。

## 防止重复计算

优先使用功能级 Buff。如果需求太粗，才使用项目级 Buff。不要对同一范围同时应用功能级 Buff 和项目级 Buff。
