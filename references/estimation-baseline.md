# 估算基准

## 端类型

- iOS
- Android
- 管理后台Web
- H5
- 小程序
- PC Web
- 服务端
- 开放API

## 角色

- 产品
- UI
- iOS
- Android
- Web前端
- H5前端
- 小程序前端
- 后端
- 测试
- 项目管理

## 计算公式

```text
基础人天 = 角色基准人天 * 标品覆盖系数 * 复杂度系数 * 风险系数
Buff人天 = 基础合计人天 * Buff比例
最终人天 = 基础合计人天 + Buff人天
统一单价报价 = 最终人天 * 人天单价 * 报价系数
角色单价报价 = sum(角色最终人天 * 角色人天单价) * 报价系数
```

默认报价系数为 `1.2`。如果用户只提供统一人天单价，所有角色都按统一单价计算。如果用户提供角色单价，按角色分别计价。

## 默认角色占比

用于需求非常粗时的拆分参考：

- 产品：8%-12%
- UI：6%-12%，仅在需要设计时计入
- 后端：30%-40%
- 各端前端合计：25%-40%，多端项目需要分端拆开
- 测试：15%-25%
- 项目管理：5%-10%

## 多端估算原则

不要把 iOS、Android、管理后台Web、H5、小程序、PC Web 合并为一个前端字段。客户主表按端列出人天，内部明细按端类型和角色拆分。

如果使用跨端框架，可以降低对应端的系数，但必须说明依据，例如“Flutter 复用 UI 和部分业务逻辑，Android/iOS 端各按 0.7 估算”。
