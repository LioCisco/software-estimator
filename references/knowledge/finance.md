# 财务系统估算知识库

## 适用范围

适用于应收应付、凭证、账簿、报表、对账、费用和资产等财务管理功能。不替代会计制度设计，复杂核算、税务、集团合并报表需要单独评估。

## 常见一级模块

- 应收应付
- 凭证
- 账簿
- 报表
- 对账
- 费用
- 资产
- 基础资料

## 模块估算基准

### 应收应付

常见二级功能：
- 应收单
- 应付单
- 收款登记
- 付款登记
- 核销
- 逾期提醒

默认复杂度：
- 台账登记：M
- 核销和对账：H
- 多币种/多组织：H

基准工时：
- 产品：1.5-3
- Web后台：3-6
- 后端：4-8
- 测试：2-4

客户确认问题：
- 是否需要自动生成应收应付？
- 是否需要核销、坏账、逾期提醒？

### 凭证与账簿

常见二级功能：
- 凭证录入
- 凭证审核
- 凭证过账
- 总账
- 明细账
- 日记账

默认复杂度：
- 手工凭证：M
- 自动凭证和多账簿：H

基准工时：
- 产品：2-4
- Web后台：4-8
- 后端：6-12
- 测试：3-6

常见风险：
- 会计科目、辅助核算、期间结账规则不清楚
- 自动凭证模板需要大量业务规则

客户确认问题：
- 是否需要自动生成凭证？
- 会计科目和辅助核算是否已定义？
- 是否需要期间结账和反结账？

### 财务报表与对账

常见二级功能：
- 余额表
- 利润表
- 资产负债表
- 银行对账
- 业务对账

默认复杂度：
- 固定报表：M
- 可配置报表和跨系统对账：H

基准工时：
- 产品：2-4
- Web后台：3-6
- 后端：5-10
- 测试：3-6

客户确认问题：
- 报表口径是否已有模板？
- 是否需要从外部系统导入对账数据？
