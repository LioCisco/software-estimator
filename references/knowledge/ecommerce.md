# 电商行业估算知识库

## 适用范围

适用于 B2C 商城、会员商城、小程序商城、H5 商城、分销商城和带营销/订单/支付/售后的交易系统。不适用于超复杂交易撮合、跨境清关或大型平台级 marketplace，除非单独评估。

## 常见一级模块

- 商品
- 订单
- 会员
- 营销
- 支付
- 物流
- 售后
- 分销
- 装修
- 报表

## 模块估算基准

### 商品管理

常见二级功能：
- 商品新增/编辑
- SKU/规格
- 商品分类
- 商品上下架
- 商品图片
- 批量导入导出

默认复杂度：
- 普通商品：M
- 多规格 SKU：H
- 多仓库存联动：H

默认端：
- 管理后台Web
- H5/小程序/PC Web 按项目范围

基准工时：
- 产品：0.5-1.5
- UI：0.5-1
- Web后台：1.5-3
- H5：1-2
- 小程序：1-2
- 后端：2-5
- 测试：1-2

常见风险：
- SKU、库存、价格体系未明确
- 是否支持多门店、多仓、多单位
- 是否和 ERP/WMS 对接

客户确认问题：
- 是否存在多规格 SKU？
- 库存是否参与下单扣减？
- 是否需要商品批量导入导出？

### 订单与支付

常见二级功能：
- 购物车
- 下单
- 订单列表
- 订单详情
- 支付
- 取消订单
- 退款
- 对账

默认复杂度：
- 普通下单：M
- 支付退款对账：H
- 拆单、合单、预售：H

基准工时：
- 产品：1-2
- UI：0.5-1.5
- 各端前端：2-5
- 后端：4-8
- 测试：2-4

常见风险：
- 支付渠道和退款规则未明确
- 优惠券、积分、运费、库存扣减顺序影响订单金额

客户确认问题：
- 支付方式有哪些？
- 是否需要退款、部分退款、自动对账？
- 是否存在优惠叠加规则？

### 营销与会员

常见二级功能：
- 优惠券
- 满减满折
- 积分
- 会员等级
- 分销返佣

默认复杂度：
- 单一优惠券：M
- 多优惠叠加：H
- 分销返佣：H

基准工时：
- 产品：1-2
- UI：0.5-1
- Web后台：2-4
- H5/小程序：1.5-3
- 后端：3-7
- 测试：2-4

客户确认问题：
- 优惠是否可以叠加？
- 分销层级和结算周期是什么？
